# AI Workflow Setup

### Objective

This repository documents the setup process completed as part of a hiring assessment. The objective was to install the required AI tools, configure the development environment, and document the process.

---

### Tools Installed

- GitHub
- Cursor IDE
- Claude Code
- OpenAI Codex

---

### Steps Completed

1. Reviewed the assessment instructions.
2. Verified the company, software, and provided links before installing anything.
3. Created a GitHub account and a public repository.
4. Created a Cursor account and installed Cursor IDE.
5. Installed the Claude Code extension.
6. Installed the Codex extension.
7. Created this README.
8. Committed and pushed the repository to GitHub.

---

### Research Process

Before installing any software, I verified that the request was legitimate by checking the company, reviewing the official websites, and comparing the hiring process with publicly available information.

During the setup, I used ChatGPT to help organize the workflow and explain unfamiliar steps. I also used Google Search, Google's AI Overview, YouTube, and official documentation to verify information and troubleshoot issues.

Before moving on to each step, I manually checked the information to ensure everything was accurate and that no mistakes had been made.

---

### Issues Encountered

Cursor initially opened in the **Cursor Agents** view instead of the standard editor. Because of this, the **Extensions** menu mentioned in the instructions was not visible.

I searched for the issue using Google Search, Google's AI Overview, and YouTube. Most of the available information referred to the updated interface, which initially made it unclear where extensions could be managed.

After approximately ten minutes of troubleshooting, I restarted Cursor and opened a new workspace. This time the standard editor loaded correctly, displaying the full sidebar and the Extensions menu. From there, I installed Codex without any further issues.

---

### Takeaways

This exercise reinforced the importance of verifying information before taking action.

Rather than relying on a single source, I compared information across multiple sources, confirmed that the request was legitimate, and validated each step before proceeding. This is the same approach I generally apply when researching information or making marketing decisions.
